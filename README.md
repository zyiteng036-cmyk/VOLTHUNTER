## 💻 プロジェクト概要

* **ゲーム概要:** 攻撃をジャストでかわして切り返す 電脳世界を駆ける切り返し爽快アクション！
* **コンセプト:** 最大のピンチを最高のリターンにしろ！
* **開発環境:** Unreal Engine 5 (C++)
* **制作期間:** 2025年11月中旬～2026年3月上旬（約4ヶ月）
* **チーム人数:** 4人（プログラマー2人、プランナー1人、デザイナー1人）
* **担当:** * 【プログラム】プレイヤー・カメラ制御全般、UI実装（設定/Clear画面）、Spline移動（床配置・電気移動）、ボス追尾攻撃AI、リスポーン機能
  * 【デザイン・アート】Niagaraエフェクト作成（一部除く）、設定画面UI、床・壁のStaticMesh作成

## 💡 技術的なこだわり・アピールポイント

本作はアクションゲームとしての「直感的な手触り」と「チーム開発における拡張性の高さ」を両立するため、単なるフラグ管理ではなく、**C++によるコンポーネント指向の設計と数学的アプローチ（空間ベクトル演算）**をコアシステムに組み込んでいます。

### 1. コンポーネント分割とGASを用いた堅牢なステート管理
* **【課題】** 攻撃、回避、移動など複雑なステート遷移を単一のCharacterクラスで管理すると、フラグがスパゲッティ化し、チーム開発においてバグの温床になる問題がありました。
* **【解決】** プレイヤーの機能を `Attack`, `Evasive`, `Movement`, `ElectroGauge` 等のコンポーネントに完全分割しました。さらに、ステート遷移のコアにはGameplay Ability System (GAS) を導入し、各アクションを安全にカプセル化しています。
* **【結果】** アクションの追加や仕様変更に強く、他メンバーが触っても意図しないバグが起きにくい、クリーンで拡張性の高いアーキテクチャを実現しました。

### 2. 先行入力バッファとAnimNotifyによる「手触り」の極限追求
* **【課題】** ハイスピードな戦闘において、ボタン連打による意図しないモーションの暴発を防ぎつつ、滑らかなコンボ連携を実現する必要がありました。
* **【解決】** アニメーションの特定フレームでのみ次アクションの予約を受け付ける「先行入力バッファ」をC++で実装しました。同時に、ヒットストップやカメラシェイクをAnimNotifyStateと緻密に連携させています。
* **【結果】** プレイヤーの思考を阻害しない直感的でレスポンスの良い操作性を実現し、アクションゲームの根幹である「最高の手触り」を生み出しています。

## 🎮 操作方法 (Xboxコントローラー推奨)

本作はゲームパッドでのプレイを想定して直感的に操作できるよう調整されています。

| アクション | 入力ボタン (Xbox準拠) |
| :--- | :--- |
| **移動 / 選択** | 左スティック |
| **カメラ操作** | 右スティック |
| **ロックオン** | 右スティック押し込み |
| **ジャンプ / 決定** | A ボタン |
| **回避 / キャンセル** | B ボタン |
| **弱攻撃** | X ボタン |
| **強攻撃** | Y ボタン |
| **スキル１（追尾電撃）** | RB ボタン |
| **スキル２（高速突進）** | RT ボタン |
| **スキル３（超広範囲攻撃）** | LB ボタン |
| **設定画面** | メニュー ボタン |

## 🚀 インストールと実行方法

※ 本作の実行には Windows 環境が必要です。

1. GitHubの **[Releases]** ページから `Windows Build v1.0` (zipファイル) をダウンロードします。
2. ダウンロードした zip ファイルを任意のフォルダに展開します。
3. 展開したフォルダ内にある実行ファイル（`.exe`）を起動するとゲームが開始されます。
